# Taystfuhl MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build, push, and deploy a Taystfuhl MVP that lets a user paste a cooking-video URL, see an attributed AI-style recipe draft, and download the recipe text.

**Architecture:** Next.js app with one public page, one decode API route, and focused recipe extraction helpers. MVP uses URL metadata plus deterministic recipe templates so the deployed frontend is testable before paid AI/video integrations are wired in.

**Tech Stack:** Next.js, React, TypeScript, CSS modules/global CSS, GitHub CLI, Vercel CLI.

---

## File Structure

- `package.json`: project scripts and dependencies.
- `app/layout.tsx`: app metadata and shared shell.
- `app/page.tsx`: client-side URL submission UI, result preview, and download behavior.
- `app/globals.css`: responsive visual system.
- `app/api/decode/route.ts`: API endpoint for URL validation and recipe draft response.
- `lib/decode.ts`: platform detection, oEmbed metadata lookup, recipe draft generation, confidence labels.
- `docs/superpowers/specs/2026-06-07-taystfuhl-mvp-design.md`: approved design spec.
